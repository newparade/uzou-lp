## コンセプト

### コンセプトワード: 「接続の闇市場」

> デジタル広告市場は「有象無象」だ。形のある広告主と、形のないインプレッション。形のあるメディアと、形のないユーザーの注意。UZOUは、この混沌に秩序をもたらすために生まれた接続者（コネクター）ではない。UZOUは混沌そのものを飼いならし、その内側で精密な循環を設計するプラットフォームだ。

「接続の闇市場」という言葉は挑発的に聞こえる。それが意図だ。広告業界の「闇」——不透明な取引、見えないマージン、検証できないロジック——に正面から向き合い、その内側で完全に可視化された循環を作る。それがUZOUの存在理由である。

**なぜこのコンセプトワードか:**
- 「闇市場」は混沌と不可視性を象徴する → UZOUが解決すべき課題の本質を一語で表す
- 「接続」はUZOUの機能的価値（広告主×メディアのマッチング）を示す
- 競合は「最適化」「効率化」を謳う。UZOUは「闇を制する者」として立つ

### 世界観

| 要素 | 定義 |
|---|---|
| 空間 | 深海の探査潜水艦の制御室。外は光が届かない暗闇だが、計器は全て正確に動き、接続ラインが生きている |
| 時間 | 静止に近い流れ。しかし止まってはいない。データは常に動いており、接続は毎秒更新されている |
| 質感 | ガラスと海水の中間。透明だが重さがある。光を屈折させながら通す |
| 温度 | 表面は冷たい（#040404の深海）。しかし接続が起きる瞬間だけ、ティールの光が灯る |
| 音 | 深海のソナー。長い沈黙の後、ピンッという接続音。そしてまた沈黙 |

**世界観の補足——「深海」をなぜ選ぶか:**

深海は可視性がゼロの世界だ。しかしその深海の中で、生命は独自のエコシステムを構築している。発光生物は接続を作り、食物連鎖は循環する。UZOUの広告市場も同様——多くのプレイヤーには「見えない」場所で、UZOUは既に500のメディアとの生態系を構築し、50億の接続を毎月処理している。

「深海」は「暗い・怖い」ではなく「見えているものが全てではない」の比喩だ。

### デザイン原則

**1. 暗闇は欠如ではなく、余地である**

深い黒（#040404）と深海ティール（#0a1215）は「何もない」背景ではなく、「接続が起きる余地」として機能させる。ダークセクションは空虚ではなく、張り詰めた静寂として演出する。奥行きを作ることで、光（ティール）が生きる。

適用: Hero背景、Solution、Results、Final CTAのダークセクション全て

**2. 接続線は生きている**

UZOUのビジュアルモチーフ（接続線・ノード）は「図解」として描かない。「現在進行中のプロセス」として描く。線は常にどこかが光っており、ノードは脈を打っている。静止画でも「今この瞬間も動いている」という生命感を持たせる。

適用: Hero Canvas、About Platform Diagram、Features内のアニメーション全て

**3. コピーは問い、デザインは答える**

テキストコピーは問いかけの形を持つ。「なぜ広告は嫌われるのか」「500のメディアが選んだのはなぜか」。その問いに対してビジュアルが応答する構造を作る。コピーとデザインが独立した要素ではなく、対話している状態。

適用: 全セクションのH2コピー設計

## リサーチ結果

### 競合クリエイティブ分析

#### b→dash（https://bdash-marketing.com/）
**ビジュアル言語**: ダーク系デザイン、データビジュアライゼーションを前面に。「データ統合」を概念図で訴求。
**強み**: データの流れをビジュアル化している点、プレミアム感
**弱み**: 競合が多く「もうひとつのデータ基盤」に見える。コピーが機能説明に終始している
**差別化チャンス**: UZOUは「データ基盤」ではなく「広告主×メディアの接続者」という独自ポジション。b→dashは内部のデータフローを訴求するが、UZOUは「外部の接続ネットワーク」を訴求できる

#### SmartNews Ads（https://ads.smartnews.com/）
**ビジュアル言語**: 明るく清潔なUI。ブルー/ホワイト基調。プロダクトUIを前面に出す
**強み**: 媒体の信頼性（SmartNewsというメディアブランド）を背景にした説得力
**弱み**: 媒体主向けのプラットフォーム訴求が弱い。広告主一辺倒のメッセージング
**差別化チャンス**: UZOUは「広告主とメディア主、双方に対して等しく価値を持つ」という二面性。SmartNews Adsにはこの構造がない

#### The Trade Desk（https://www.thetradedesk.com/）
**ビジュアル言語**: エンタープライズ向けの重厚な訴求。グローバルスケールを感じさせるビジュアル（地球規模のネットワーク）
**強み**: スケールと信頼感。「世界最大のプログラマティックバイサイドプラットフォーム」という権威
**弱み**: グローバル企業向けで、日本の中小媒体社や代理店に「自分事化」しにくい。クリエイティブが同質的
**差別化チャンス**: UZOUは「日本のWebメディアエコシステム」に特化した深さ。「500のメディアが選んだ」という具体性がTTDにはない

**総合差別化ポイント:**

競合3社全てが「プラットフォームの機能」を訴求している。UZOUは「接続そのものの哲学」を訴求する。競合は「何ができるか」を語る。UZOUは「なぜこれが必要か」を語る。その上で「500のメディア」という具体的な接続実績が証拠になる。

### トレンド・文化的文脈

#### AdTech業界のデザイントレンド（2025-2026）
- データビジュアライゼーションのUI化：リアルタイムダッシュボード風の表現が主流
- ダークUI採用の加速：「プレミアム・先進性」の記号として定着
- ネットワーク図解：グラフ構造（ノード&エッジ）をビジュアルメタファーに使う事例が増加
- マイクロインタラクション強化：静的なページをやめ、「生きているデータ」として表現

**UZOUが取り入れるべき点**: ネットワーク図解 × ダークUI。ただし競合全員がやっている「データの量」訴求ではなく、「接続の質」を視覚化する方向性へ。

#### ターゲット層（広告代理店メディアバイヤー / 30-40代）が親しんでいるビジュアルカルチャー
- Linear、Vercel、FigmaなどのデベロッパーツールのUIデザイン感覚を持つ
- 「機能説明よりも世界観」で製品を判断するデザインリテラシーを持つ
- 「AIが〜」「次世代〜」という言葉に免疫を持ち、むしろ嫌悪感を持つ層
- 実績数値と導入企業名の具体性を重要視する

**UZOUが取り入れるべき点**: LinearやVercel級の「洗練されたダークUI + 精密なモーション」という質感。ただしデベロッパーツールのクールさではなく、AdTech固有の「接続・フロー・循環」の美学を加える。

#### 「有象無象」の哲学に共鳴する文化的参照
- **建築**: ルイス・カーンの「沈黙と光」（暗闇から光が生まれる建築哲学）
- **映画**: ダークマター（暗黒物質）を扱うSF——「見えないものが宇宙の大半を占める」
- **音楽**: アンビエントテクノ（Brian Eno、Autechre）——沈黙の中に複雑な構造が存在する
- **日本の美意識**: 「間（ま）」の概念——何もない空間が最も意味を持つ

これらが示すのは、「暗闇・空白・見えないもの」は欠如ではなく、それ自体が意味と価値を持つという思想だ。UZOUの深海ティールはこの系譜に連なる。

## Heroコピーの決定

### 最終採用コピー

**メインコピー（H1）: 「広告が、メディアを救う。」**

採用理由:
- 7語以内の制約を満たす（6語）
- 逆説的な強さを持つ。「広告は嫌われる存在」という先入観を一撃で裏返す
- UZOU固有の哲学（広告主×メディア双方への価値創出）を圧縮している
- 競合が絶対に使えない言葉。「広告が嫌われている」という現実を認めた上で「救う」と言えるのは、500のメディアとの実績を持つUZOUだけ
- 「メディアを救う」という主語（広告）が予想外。スキャンが止まる

**拒否した理由（他の案）:**

案B「500のメディアが選んだ、広告の答え。」:
社会的証明として有効だが、コピーとして記憶に残らない。「答え」という言葉が抽象的。見出しの役割（最初の衝撃）を果たせない。数値はサブコピーで使う方が効果的。

案C「有象無象の広告市場に、秩序をもたらす。」:
哲学的だが、ターゲット（メディアバイヤー）に「自分のビジネス課題が解決する」という直接的なベネフィットが伝わらない。「秩序」という抽象語も弱い。

---

### サブコピー（決定版）

```
500を超えるメディアが選んだ、接続の力。
広告主の成果と、メディアの収益を、同時に育てる。
```

設計意図:
- 1文目: 社会的証明（500メディア）× UZOUの本質（接続）
- 2文目: 双方向の価値創出を「育てる」という有機的な動詞で表現
- 「最適化」「最大化」「次世代」は全て排除

### カラー哲学

#### カラーシステムの再定義

単に「5色のブランドカラーを使う」のではなく、それぞれが「役割」と「物語上の意味」を持つ。

| 変数名 | HEX | 物語上の意味 | 使用ルール |
|---|---|---|---|
| --c-abyss / `#040404` | 深海の底 | 一切の光が届かない場所。しかしUZOUはここに接続ラインを走らせる | ページ全体の基底色。何も存在しない空間として使う |
| --bg-base / `#0a1215` | 深海 | UZOUのホーム。全ての活動が起きる場所 | Hero・ダークセクションの主背景 |
| --c-dark / `#1F353E` | 深淵 | 深海の中で形成された「地形」。UZOUのプラットフォームが構造化されている場所 | カードの背景、グラスモーフィズムのベース |
| --c-deep / `#2B4954` | フロー | 接続が通る水路。データが流れる経路 | グラデーションの中間色、接続線の色 |
| --c-primary / `#34626F` | 接触点 | 広告主とメディアが出会う瞬間の色。CTAはここで止まれ | 主要CTA、リンク、インタラクティブ要素 |
| --c-light / `#8BC0CA` | 光 / 成果 | 接続が成功したとき、深海に灯る光。数値・実績・グロー効果はこの色 | ハイライト、グロー、数値強調、ホバー状態 |

#### カラー使用の禁則事項

- ライトセクション（About-About関連）で `#8BC0CA` をテキストに使わない（コントラスト不足）
- `#040404` の上に `#1F353E` のテキストを置かない（視認性ゼロ）
- グラデーションは常にこの方向で: `深→浅（#040404 → #8BC0CA）`。逆方向は使わない
- アクセントカラーの多用を禁ずる。`#8BC0CA` の光は「少ないから美しい」

#### スクロールによるカラーストーリーの体験設計

```
Hero（#040404 / #0a1215）— 深海の底から始まる
  ↓ マーキー遷移（グラデーションで徐々に浮上）
Media Trust（#1F353E）— 実績の重さを感じる深淵
  ↓ タブ遷移で「課題」を体感
```

### タイポグラフィディレクション

#### 5段階ヒエラルキー（厳守）

```
Display: Inter Black 900 / 72-96px / letter-spacing: -0.05em
  用途: Hero H1のみ。「広告が、メディアを救う。」
  
H1:      Inter ExtraBold 800 / 52-64px / letter-spacing: -0.04em
  用途: Final CTAのH2など、最も重要なセクション見出し
  
H2:      Inter Bold 700 / 36-44px / letter-spacing: -0.03em
  用途: 全セクションの見出し
  
Body:    Noto Sans JP Regular 400 / 16px / line-height: 1.9
  用途: 全ての本文テキスト
  
Caption: Inter Medium 500 / 12-13px / letter-spacing: 0.12em / uppercase
  用途: セクションラベル（THE PROBLEM、HOW IT WORKS等）
```

#### タイポグラフィの「重さ」哲学

**コントラストのルール**: Display（900）の直後には必ずBody（400）を配置する。極太と極細の対比が「知的な挑発」の質感を作る。中間のウェイト（600/700）は見出し内のアクセントに限定し、本文周辺に出しすぎない。

**ダークセクションでのタイポ**: ダーク背景では `color: #e2e8f0`（オフホワイト）を使う。純白（#ffffff）は使わない。純白はグロー効果に使う特別色として温存する。

**グラデーションテキスト（制限的使用）**: `background-clip: text` によるグラデーションテキストは、Resultsセクションの主要数値（「50億+」等）にのみ使用する。全ての数値に使うと陳腐化する。

### モーションディレクション

#### 動きの「人格」

UZOUのモーションは**「精密だが有機的」**。機械的な等速度運動は使わない。かといって過度にバウンシーでもない。

参照すべき動きの質感: 深海生物の動き——目的を持って、しかし水の抵抗を受けながら、滑らかに移動する。

#### イージング仕様

```css
/* スクロールリビール（全要素）*/
--ease-reveal: cubic-bezier(0.22, 1, 0.36, 1);
/* 説明: 最初は速く入り、最後は非常にゆっくり止まる。有機的な減速 */

/* ホバーイン */
--ease-hover-in: cubic-bezier(0.4, 0, 0.2, 1);

/* ホバーアウト（より遅く）*/
--ease-hover-out: cubic-bezier(0.2, 0, 0, 1);

/* Canvasパーティクルの接続線描画 */
--ease-connect: cubic-bezier(0, 0, 0.2, 1);
/* 説明: 最初はゆっくり始まり（接続を探している）、急加速して到達 */
```

#### アニメーションパターン別方針

**フェードアップ（全セクション標準）**: `translateY(32px) → 0` + `opacity: 0 → 1`、duration: 0.7s、ease: `--ease-reveal`

**カウントアップ（Results数値）**: ブラーリビール（`blur(8px) → 0`）を同時に使用。「霧の中から数字が現れる」演出。duration: 1.2s

**接続線描画（About Platform Diagram）**: SVGの `stroke-dashoffset`。ただし左→右ではなく、中心のUZOUノードから外側に向かって「伸びる」方向で描画。

**ノードのパルス（Hero Canvas）**: `scale(1) → scale(1.3) → scale(1)` + グロー強度変化。3秒に1回、ランダムなノードが「脈打つ」。人間の心拍数に近いリズムで設定。

**セクション遷移のグラデーション溶解**: 背景色の切り替えは瞬時に行わない。`mask-image: linear-gradient` を使ったグラデーション溶解で、「深海を浮上していく」体験を作る。

**マウス追従（Hero Canvas）**: ホバーポイントから200px以内のノードが「引き寄せられる」。ただしノードが完全にマウスに追従するのではなく、引力に抵抗しながら近づく動き（バネ係数: 0.05）。

## 技術・ライブラリ提案

### 推奨技術（コンセプト実現の順番に）

**1. Vanilla Canvas API（Hero Connection Visualizer / 必須）**

「広告主×メディアが出会う瞬間」を視覚化する中核技術。外部ライブラリは一切不要。コンセプト（接続）と技術（Canvas描画）が完全に一致しており、技術選定の迷いはない。

実装指針:
- `requestAnimationFrame` + 30fps制限（`performance.now()` によるフレームスキップ）
- ノード描画: `arc()` + `createRadialGradient()` でグロー効果
- 接続線描画: `bezierCurveTo()` でベジェ曲線（直線は使わない）
- マウスインタラクション: `mousemove` イベント + 距離計算（バネ係数でバッファ）
- `prefers-reduced-motion` チェック: 検出時はアニメーションを停止、静止画状態に

**2. IntersectionObserver API（スクロールアニメーション / 必須）**

全セクションのリビールアニメーション。`threshold: 0.15` で早めに発火させ、遅延（80ms stagger）で順次表示。

**3. SVG stroke-dashoffset animation（About Platform Diagram / 必須）**

CSSアニメーション + IntersectionObserverのトリガーで、スクロール進入時に接続線が描かれる演出。

**4. CSS Custom Properties + backdrop-filter（ガラスモーフィズム）**

Header・Testimonialカードに使用。`backdrop-filter: blur(24px)` + `rgba()` による半透明背景。

**5. CSS scroll-driven animations（実験的、オプション）**

`animation-timeline: scroll()` は最新ブラウザのみ対応。使用する場合は `@supports` でフォールバック必須。カラーストーリー（深海→浮上）の実装に使えるが、互換性リスクを考慮してIntersectionObserver版をメインに。

### 外部ライブラリ方針

**技術制約: CDN一切不使用。Vanilla JS/CSS/HTML のみ。**

この制約は制限ではなく、哲学の体現だ。「他の何にも依存しない」技術設計が、UZOUの「独自商流基盤」というブランドメッセージと一致する。

### 技術的リスク・注意点

| リスク | 対策 |
|---|---|
| Canvas パフォーマンス（特にモバイル）| ノード数をモバイルで8個以下に削減。`devicePixelRatio` に注意 |
| `backdrop-filter` Safari対応 | `-webkit-backdrop-filter` のベンダープレフィックスを必ず付与 |
| ダークUI+ティールのコントラスト不足 | `#8BC0CA` 上の `#040404` テキストは使わない（コントラスト比4.5以下） |
| SVG接続線のスムーズ描画 | `stroke-linecap: round` と `stroke-linejoin: round` で曲線を滑らかに |
| `prefers-reduced-motion` への対応 | 全Canvasアニメーション、CSS transition/animation に `@media (prefers-reduced-motion)` を適用 |

## art-directorへの引き継ぎ

### ビジュアルで実現してほしいこと

1. **「深海から光へ」の体験がスクロールで感じられること**: ページを下に読み進めるにつれ、視覚的な「明るさ」が増す感覚。ダーク（深海）→ライト（浮上）→ダーク（決意）→ライト（成果）のリズムを設計する

2. **Heroの右半分が「生きていること」**: UZOUの接続ビジュアライザーは、見た人が「これは何をしているんだろう」と3秒間見続けてしまうクオリティを目指す。パーティクルではなく「接続の実況中継」として見える

3. **コピーとビジュアルの対話**: 「広告が、メディアを救う。」というコピーが出てくる瞬間、その右側のCanvasで接続線がちょうど完成する演出を作れるか検討してほしい

4. **ダークセクションの奥行き**: `#0a1215` の単色にしない。`radial-gradient` × ノイズテクスチャ × アンビエントグローの3層で奥行きを作る

5. **「選ばれる理由」セクションのビジュアル**: 4つのカードが「均等な説明」に見えないよう、大カード1枚 + 小カード3枚の非対称ベントグリッドを維持する

### 絶対に守ってほしいルール

1. 接続線・ノードは「幾何学的・線形」に保つ。有機的なブロブ・バブル・カーブは使わない
2. `#8BC0CA`（ライトティール）は本当に重要な要素にのみ使う。全体の5%が上限
3. コピーで決定した「接続」「流れ」「深海」のメタファーをデザインが裏切らないこと。「スマートに」「最適化」のビジュアル言語（スタイリッシュな青×白）には絶対に行かない
4. 全アニメーションに `prefers-reduced-motion: reduce` 対応を実装する
5. Hero H1「広告が、メディアを救う。」は Display フォント（Inter 900 / 72px以上）で表示する

### 自由に判断してよい範囲

- 各セクションの具体的なカードレイアウト（コンセプトを守っていれば自由）
- グラスモーフィズムの適用範囲（Header・Testimonialsは必須だが、他セクションへの展開は判断に委ねる）
- モバイル時のビジュアル簡略化（Canvas ノード数、接続線の太さなど）
- セクション遷移のグラデーション具体的な実装方法
- Feature内の小アニメーションの具体的な表現（「バーチャートが変動する」という方向性は守ること）
