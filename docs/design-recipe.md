# UZOU LP v9 デザインレシピ

## リサーチサマリー

### 分析サイト数
- ギャラリーから30サイト収集（Tier 1: 5, Tier 2: 7, Tier 3: 18）
- AdTech競合LP 5サイト詳細分析（b→dash, Logicad, SmartNews Ads, Criteo, The Trade Desk）
- ナレッジベースから22サイトのパターン参照

### 横断分析から見えた傾向

**BtoB SaaS / AdTech LPの主流パターン:**
1. 白ベース背景 + ブランドカラーのアクセント
2. Noto Sans JP + 欧文Webフォント（Inter / Poppins）の組み合わせ
3. 48px以上の大型ヘッドライン
4. 3連CTAブロック（資料DL / 問い合わせ / デモ）が最も効果的
5. セクションラベルに英字キャプション（"Features", "Results"等）
6. 導入実績数値 + クライアントロゴ + 受賞バッジの3層信頼構築

**差別化ポイント:**
- ティール系カラーはBtoB SaaS界隈では珍しい（大半がブルー/パープル）→ 独自性になる
- 「有象無象」コンセプトを幾何学的装飾で表現 → 他社にない世界観
- 広告主×媒体主の2軸ターゲット構成 → Criteoのタブ切替手法が参考に

## デザイン方向性提案: 3案

---

### 方向性A: 「Clean Premium」（白ベース×ティール）

**コンセプト**: 信頼感と先進性を白の清潔感で表現。余白を贅沢に使い、ティールカラーがアクセントとして映えるプレミアムなBtoB LP。

**参考イメージ**: KARTE + The Trade Desk + バクラク

**カラー設計**:
- 背景: #FFFFFF / #F0F7F8（ティール系薄背景） / #E8F2F4
- テキスト: #040404（見出し） / #4a5568（本文） / #718096（補助）
- アクセント: #34626F（CTA / リンク） / #8BC0CA（ホバー / ハイライト）
- ダークセクション: #1F353E（Solution / Final CTA）

**タイポグラフィ**:
- 見出し: Inter 800 / 48-60px / letter-spacing: -0.03em
- 数値: Inter 800 / 56px
- 本文: Noto Sans JP 400 / 15-16px / line-height: 1.9
- セクションラベル: Inter 600 / 13px / letter-spacing: 0.15em / uppercase

**レイアウト**:
- コンテンツ幅: 1200px
- セクション間余白: 120-160px
- 1カラム基調、Feature部分のみ左右交互2カラム
- カード: border-radius: 20px / subtle shadow

**装飾**:
- ドットグリッド背景（Heroセクション）→ Linear参考
- セクション間は背景色の切替のみ（波形/斜め不使用）
- SVGアイコンは線（stroke）スタイルで統一
- ティール系の微グラデーション（グラスモーフィズムは使わない）

**動き**:
- IntersectionObserverベースのフェードイン
- カウントアップ（Results数値）
- ホバーでカード微浮上 + shadow拡大
- スクロールプログレスバー

**向いているシーン**: 信頼重視、堅実なBtoB印象を優先したい場合
**トレードオフ**: ややコンサバで「おっ」ポイントが少ない可能性

### 方向性B: 「Geometric Flow」（幾何学×流動的モーション）

**コンセプト**: 「有象無象」＝形あるものとないものが溶け合うメタファーを、幾何学的パターンと流動的なアニメーションで表現。テック感と独自性を両立。

**参考イメージ**: Linear + Stripe Sessions + ノバセル

**カラー設計**:
- 背景: #FFFFFF → #1F353E（セクションカラースイッチ）
- テキスト: #040404 / #e2e8f0（ダーク時）
- プライマリ: #34626F（グラデーション始点）→ #8BC0CA（終点）
- アクセント: #2B4954
- グラデーション: linear-gradient(135deg, #34626F, #8BC0CA)

**タイポグラフィ**:
- 見出し: Inter 900 / 52-64px / letter-spacing: -0.04em（タイト）
- 数値: Inter 800 / 60px + グラデーションテキスト（background-clip: text）
- 本文: Noto Sans JP 400 / 16px / line-height: 1.8
- キャッチコピー: Noto Sans JP 900 / 40px

**レイアウト**:
- コンテンツ幅: 1200px
- セクション間: 100-140px
- ヒーロー: 左テキスト + 右にCanvasパーティクル（幾何学的ノード）
- Feature: ベントグリッド（不等サイズのカード配置）
- Solution: フルワイドダーク背景 + 2カラム
- セクション間で白→ダーク→白のリズム（Apple / Stripe式）

**装飾**:
- **Canvasパーティクル**: Heroにノード＆ラインの幾何学的パーティクル（広告×メディアの接続を暗喩）
- **グリッドライン背景**: 薄いグリッド線で精密感
- **ティールのアンビエントグロー**: ダークセクションで光の演出
- **ノイズテクスチャ**: ダークセクションに微妙な粒子感
- SVGアイコン: ストロークドローアニメーション付き

**動き**:
- Canvasパーティクル（Hero常時アニメーション）
- スクロール連動フェードイン + スタッガー（時間差）
- カウントアップ + 数値グラデーション
- ボタンのマグネティック効果
- セクション進入時の背景色トランジション
- テキストスタガーアニメーション（見出し1文字ずつ表示）

**向いているシーン**: テック感・先進性をアピール、「おっ」ポイントを作りたい場合
**トレードオフ**: アニメーション量が多く、実装コスト高め。パフォーマンス注意

### 方向性C: 「Data-Driven Elegance」（データビジュアル×上品さ）

**コンセプト**: 広告プラットフォームとしてのデータ・AI能力を、ダッシュボード風のビジュアルとエレガントなデザインで表現。b→dash的な情報設計 × The Trade Desk的なプレミアム感の融合。

**参考イメージ**: b→dash + Superhuman + Amplitude

**カラー設計**:
- 背景: #FFFFFF / #F8FAFB（ほぼ白のグレー）
- テキスト: #040404 / #4a5568
- プライマリ: #34626F
- グラデーション背景（Hero）: linear-gradient(160deg, #f8fcfd, #E8F2F4, #d4edf2)
- ダークセクション: linear-gradient(160deg, #1F353E, #1A2F38)
- チャート/データ視覚化カラー: #8BC0CA / #34626F / #2B4954

**タイポグラフィ**:
- 見出し: Inter 800 / 48-56px / letter-spacing: -0.02em
- 数値（Results等）: Inter 800 / 64px + ティールグラデーション
- 本文: Noto Sans JP 400 / 15px / line-height: 2.0（ゆったり）
- バッジ/タグ: Inter 600 / 12px / uppercase

**レイアウト**:
- コンテンツ幅: 1200px
- セクション間: 120-160px
- ヒーロー: 左テキスト + 右にダッシュボードUIモック（CSS製）
- Feature: 左右交互の2カラム + 各Feature内にミニデータビジュアル
- Results: 4カラムの大型数値カード + ミニチャートSVG
- CTA Strip: 3連カード（b→dash式のアイコン+テキスト+矢印）

**装飾**:
- **ダッシュボードモック**: HeroにCSS製の擬似UI（チャート、数値カード）
- **SVGミニチャート**: Results / Featureにデータビジュアル
- **バーチャートアニメーション**: Feature03で成長を表現
- **薄いティール背景グラデーション**: 微妙な色変化で上品さ
- 装飾は控えめ、データビジュアルそのものが装飾の役割

**動き**:
- IntersectionObserverベースのフェードイン
- チャートの描画アニメーション（SVG stroke-dashoffset）
- カウントアップ（Results）
- バーチャートの成長アニメーション
- ダッシュボードカードの順次表示（スタッガー）
- ホバーでカード shadow 拡大

**向いているシーン**: データ・AI能力の訴求、ダッシュボードの魅力を暗示したい場合
**トレードオフ**: ダッシュボードモックの作り込みが必要。v8と方向性が近くなりがち

## 推奨

**方向性B「Geometric Flow」** を推奨。

理由:
1. v8と最も異なるアプローチで「作り直し」の意味がある
2. 「有象無象」のブランドコンセプトを幾何学的パーティクルで直接表現できる
3. セクションカラースイッチ（白→ダーク→白）で視覚的リズムが生まれる
4. ティール系グラデーションがLinear / Stripe級のモダン感を出せる
5. BtoB SaaSのティール系LPは少ないため、差別化になる
6. テキストスタガー・パーティクル等の「おっ」ポイントが複数作れる

**ただし、方向性Cのデータビジュアル要素（ダッシュボードモック、ミニチャート）はB案にも部分的に取り入れたい。**

## 今回「使わない」もの（意図的に避けるテクニック）

| 避けるもの | 理由 |
|---|---|
| 丸いバブル/ブロブ装飾 | ブランド方針で「幾何学的・線形」が指定されている |
| 実際の管理画面スクリーンショット | 会議で「コンセプト訴求優先、UIは見せない」と決定 |
| 過度なイラスト | 作り込みに時間がかかり、テンプレ感が出やすい |
| カルーセル / 横スクロール | モバイルUXが悪く、コンテンツが隠れるリスク |
| Bootstrap / Material Design風のカード | AI生成感・テンプレ感の排除 |
| CDN依存のライブラリ（GSAP等） | 技術制約: Vanilla JS のみ |
