## コンセプト

### コンセプトワード: 「Precision Lattice（精密な格子）」

> 広告市場は混沌としている。無数の広告主と無数のメディアが、互いの存在を知らないまま漂っている。UZOUは、この混沌に「格子」を敷く。精密に計算された格子は、あらゆる接続の可能性を可視化し、最適な組み合わせをリアルタイムで選び出す。格子は静止画ではない。毎秒書き換えられ、進化し続ける生きた構造体だ。

**なぜ「Precision Lattice」か:**

- 「格子（Lattice）」は幾何学的であり、UZOUのGeometric Flowコンセプトと直結する
- 格子は「繰り返し」と「規則性」を持つが、その交点は無限に存在する — 500以上のメディアとの接続可能性を暗喩
- 「精密（Precision）」はAI最適化の正確さと、BtoBプラットフォームとしての信頼性を一語で表す
- 競合が「最適化」「効率化」「自動化」を謳う中、UZOUは「精密さ」という質の次元で差別化する
- v10の「闇市場」はダーク世界観への依存が強すぎた。「格子」はライトでもダークでも成立するモチーフ

**NGだった候補:**
- ~~「Neural Mesh（神経網）」~~ — AI/ニューラルネットワーク連想が陳腐
- ~~「Liquid Grid（液体の格子）」~~ — 液体メタファーはブロブ装飾を誘発
- ~~「Signal Pathway（信号経路）」~~ — 回路基板イメージがテック臭すぎる

### 世界観

| 要素 | 定義 |
|---|---|
| 空間 | 精密機器のショールーム。壁は白く、天井は高く、光が均一に満ちている。しかし展示品（接続のビジュアル）だけがティールの光を放っている。Apple Store × 管制塔 |
| 時間 | 流れている。しかし急いではいない。毎秒50億のリクエストを処理しているが、その速度を感じさせない静謐さ。時計の秒針のように、確実に前に進む |
| 質感 | 磨き上げられたガラスの上に、精密な金属の格子線が浮かんでいる。触れると冷たいが、中に光が通っている。硬質だが透明 |
| 温度 | クール寄りのニュートラル。白い空間は冷たくも温かくもない。しかし接続が成立する瞬間だけ、ティールの温かみが灯る |
| 音 | 静かなサーバールームの空調音。時折、接続成立を知らせる柔らかな電子音。背景に微かな高周波のハム |

**v10「深海」からの転換理由:**
v10の深海メタファーはクリエイティブとして強力だったが、BtoB法人向けLPとしてダークUIが支配的になりすぎた。クライアントフィードバック「ダークすぎるのはNG」に対応し、白ファーストを維持しつつ「ハイクオリティに見える」方向性へ転換する。「精密機器のショールーム」は白い空間に価値ある展示物が際立つ構造——まさに「白ベースなのにリッチ」を実現するメタファー。

### デザイン原則

**1. 白は「空白」ではなく「舞台」である**

白い背景を「何もない」状態ではなく、「精密な装飾が映える舞台」として機能させる。具体的には: 白い背景の上にグリッドライン（opacity 0.04-0.08）を敷き、ドットパターンやSVG幾何学装飾を配置し、「計算された空間」であることを常に感じさせる。SmartHRが独自イラストで達成した「テンプレート感ゼロ」を、UZOUはSVG格子装飾の精緻さで達成する。

適用: 全ライトセクション（About、Features、Testimonials、Flow、FAQ）の背景処理

**2. 装飾は3層で重ねる — 背景装飾 / コンテンツ装飾 / マイクロインタラクション**

1セクションに1種類の装飾では「平坦」に見える。各セクションに最低3層の装飾を重ねることで「リッチ」に見せる。ただし3層それぞれのopacityを厳密に制御し、互いが干渉しない設計とする。

- **Layer 1 — 背景装飾（opacity 0.03-0.10）**: グリッドライン、ドットパターン、SVG幾何学形状、グラデーションの光
- **Layer 2 — コンテンツ装飾（opacity 0.10-0.40）**: カードのグラデーションボーダー、セクションラベルの装飾線、数値のグロー
- **Layer 3 — マイクロインタラクション（opacity 0.40-1.0）**: ホバーのグロー拡大、スクロール連動のストロークドロー、ボタンのシマーエフェクト

適用: 全セクション

**3. アニメーションは「種類」で語る — 同じ動きを繰り返さない**

現状のUZOUは全セクションがフェードイン（translateY + opacity）で均一。各セクションに固有のアニメーションパターンを割り当てることで「スクロールの旅」を演出する。以下の7パターンを使い分ける:

1. **Stagger Reveal** — 要素が時間差で出現（Hero）
2. **Blur Unveil** — ぼかしが晴れて出現（Results数値）
3. **Slide In** — 左右からスライドイン（Features左右交互）
4. **Scale Up** — 小→大にスケールアップ（Testimonialカード）
5. **Stroke Draw** — SVGの線が描かれる（About Platform Diagram、Flow Timeline）
6. **Counter Cascade** — 数値がカスケードでカウントアップ（Results）
7. **Parallax Drift** — スクロールに連動した微小パララックス（背景SVG装飾）

適用: セクション別アニメーション計画（後述）で詳細割当

## リサーチ結果

### 競合クリエイティブ分析

#### 分析1: Yappli（yappli.co.jp）— 動画Hero + SVGタイポ
**ビジュアル言語**: Vimeo動画背景のフルスクリーンHero。SVGでレンダリングした「Mobile Tech for All」のタイポグラフィアニメーション。IntersectionObserverによるスクロールリビール。
**強み**: ファーストインプレッションの圧倒的な「厚み」。動画1本で「テンプレートではない」と証明。
**弱み**: Hero以降は標準的なレイアウト。動画の制作コストが高い。
**UZOUへの示唆**: 動画は使えない（Vanilla制約）。代わりにCanvas + CSSグラデーション背景の2層レイヤリングでHeroの「厚み」を出す。Yappliが「動画」で達成した視覚密度を、UZOUは「幾何学装飾の重ね合わせ」で達成する。

#### 分析2: SmartHR（smarthr.jp）— カスタムイラスト × 統一デザインシステム
**ビジュアル言語**: 独自のフラットイラストレーションがサイト全体を貫く。Astroの軽量フレームワーク上で、スタッガードフェードインが統一されている。「シェアNo.1」を前面に打ち出す実績訴求。
**強み**: テンプレート感ゼロ。イラストの統一トーンが「このブランドでしかありえない」世界観を作る。
**弱み**: イラスト制作にリソースが必要。アニメーションは控えめ。
**UZOUへの示唆**: カスタムイラストの代わりに、CSS/SVGで生成した幾何学グラフィックをUZOU固有のビジュアル言語にする。SmartHRが「人間」で温かみを出しているのに対し、UZOUは「幾何学の精密さ」で知性とプロフェッショナリズムを出す。

#### 分析3: Outbrain Japan（outbrain.com/jp/）— 大型数値カウンター × 動画Hero
**ビジュアル言語**: Wistia動画Hero。パートナー数値の大型カウンター表示（+35% / -90%）。Swiperカルーセル。ティール→パープルのTeadsブランドグラデーション。
**強み**: 数値の視覚的インパクトが強い。グローバル企業のデザイン品質。
**弱み**: KARTEと色域が近い。日本独自のデザイン表現は薄い。
**UZOUへの示唆**: Results数値の視覚インパクトをOutbrainレベルまで引き上げる。現状のUZOUは数値がやや控えめ。背景にデータフローSVGを加え「データが流れている」感を強化。

#### 分析4: KARTE（karte.io）— ティール系 × メガメニュー
**ビジュアル言語**: Gatsby SSG。#2AAB9F（ティールグリーン）をブランドカラーに。メガメニューの情報設計が秀逸。プロダクトカードグリッド。
**強み**: 多プロダクトの整理力。メガメニューの品質。
**弱み**: UZOUとティール系で色域が被る。装飾は控えめ。
**UZOUへの示唆**: KARTEが「プロダクトの多様性」で差別化しているのに対し、UZOUは「ネットワークの規模感」で差別化。色域が近い以上、差別化は**装飾の質と密度**、**アニメーションの多様性**、**グラデーションの方向性**で行う。KARTEがフラットなティールを使うのに対し、UZOUは `#34626F → #8BC0CA` のグラデーションに方向性（135deg）を持たせて差別化。

#### 分析5: Sansan（jp.sansan.com）— 多ライブラリ活用の動的体験
**ビジュアル言語**: AOS.js + Swiper + Isotope + Lightbox + jQuery。Adobe Fonts（Typekit）。多様なスクロールアニメーション。
**強み**: アニメーションパターンの多様性。5つのJSライブラリが生み出す動的な体験。
**弱み**: ライブラリ依存が重い。パフォーマンスコスト。
**UZOUへの示唆**: Sansanが5ライブラリで実現している動的多様性を、UZOUはVanilla JSの自前実装で実現する。重要なのは「動きの種類の数」。1種類のフェードインではなく、7パターンのアニメーションで各セクションの個性を出す。

**総合差別化マトリクス:**

### トレンド・文化的文脈

#### 2025-2026 BtoB LP デザイントレンド
1. **白ベース + リッチ装飾**: Apple、Stripe、Vercelが牽引する「白い空間の中のプレミアム表現」。白は「安い」ではなく「余裕がある」の記号。
2. **グラデーションボーダー**: カードの縁にグラデーションを走らせる手法。Stripe Sessions、LayerXが採用。平坦なカードに一瞬で「立体感」と「先進性」を付与。
3. **マイクロインタラクション強化**: ホバーの3D Tilt、マグネティックボタン、シマーエフェクト。Linear、Framerが定着させた「触れると反応する」UI。
4. **SVG装飾の精緻化**: ドットグリッド、幾何学パターン、ストロークドローアニメーション。背景をSVGで「呼吸」させる手法。
5. **非対称レイアウト**: 均等な3カラムグリッドはテンプレート感の象徴。不均等なベントグリッドやオフセット配置で「デザインされている」感を出す。

#### ターゲット層のビジュアルリテラシー
- 広告代理店のメディアバイヤー（30-40代）は、日常的にLinear/Figma/Notion級のUIに触れている
- 「AI搭載」「次世代」という言葉に免疫がある。言葉より視覚体験で判断する
- 実績数値と導入企業名の**具体性**を重視。美しいだけのLPは「中身がない」と判断される
- SmartHR/freee/KARTE等のBtoB SaaSの品質を「当然の基準」として持っている

### カラーディレクション

#### v11の白ファーストカラー体系

| 変数名 | HEX | 用途 | 使用比率 |
|---|---|---|---|
| `--c-base` | `#ffffff` | ページ全体のベース背景 | 40% |
| `--bg-surface` | `#f5fafb` | Featuresカード背景、Flow背景等 | 15% |
| `--bg-tint` | `#F0F7F8` | Testimonials、FAQ背景のティント | 10% |
| `--bg-tint2` | `#E8F2F4` | About背景のティント | 5% |
| `--c-dark` | `#1F353E` | ダークセクション（Solution/Final CTA） | 15% |
| `--c-primary` | `#34626F` | CTA、リンク、アクセント | 7% |
| `--c-light` | `#8BC0CA` | ハイライト、グロー、数値強調 | 5% |
| `--c-deep` | `#2B4954` | グラデーション中間色 | 3% |

**カラーの哲学的転換:**
v10は「暗闇の中の光」（ダーク80%に光5%）。v11は「光の中の精密さ」（ライト75%に精密な装飾）。光（#8BC0CA）の使用比率5%は据え置き——少ないから美しい、という原則は不変。

#### カラー使用ルール（禁則事項）
- `#8BC0CA` をライト背景上のテキストに使わない（コントラスト不足）
- ダークセクションは `#1F353E` を最暗として使う。`#040404` はテキストのみ
- グラデーション方向は135deg（左上→右下）で統一。「接続が右下へ流れる」を暗喩
- ティールの光は主要KPI数値、CTAのグロー、ホバーハイライトにのみ使用

### タイポグラフィディレクション

#### 5段階ヒエラルキー

```
Display: Inter 900 / clamp(52px, 6vw, 88px) / letter-spacing: -0.05em
  用途: Hero H1「広告が、メディアを救う。」のみ
  
H2:      Inter 700-800 / clamp(30px, 3.5vw, 44px) / letter-spacing: -0.03em
  用途: 全セクション見出し
  
H3:      Inter 600 / clamp(18px, 2vw, 24px) / letter-spacing: -0.02em
  用途: Featureカードタイトル、Flowステップタイトル
  
Body:    Noto Sans JP 400 / 16px / line-height: 1.9
  用途: 全本文テキスト
  
Caption: Inter 500-600 / 12-13px / letter-spacing: 0.12em / uppercase
  用途: セクションラベル（// THE PROBLEM 等）
```

#### タイポの「リッチさ」を出す手法
- H2見出しに `background-clip: text` のティールグラデーション——ただし全セクションではなく Results と Final CTA のみ
- セクションラベル（Caption）に左の `//` プレフィックス + アンダーラインのアクセント装飾
- 数値（50億+等）は Inter 800 + グラデーション + 微かなテキストシャドウ（`0 0 40px rgba(139,192,202,0.3)`）

### モーションディレクション

#### 動きの「人格」: 精密だが有機的

イージング仕様（変更なし、v10確立済み）:
```css
--ease-reveal:    cubic-bezier(0.22, 1, 0.36, 1);   /* スクロールリビール */
--ease-hover-in:  cubic-bezier(0.4, 0, 0.2, 1);     /* ホバーイン */
--ease-hover-out: cubic-bezier(0.2, 0, 0, 1);       /* ホバーアウト */
--ease-connect:   cubic-bezier(0, 0, 0.2, 1);       /* 接続線描画 */
```

#### 7パターンのアニメーション仕様

| # | パターン名 | CSS/JS実装 | 使用セクション |
|---|---|---|---|
| 1 | Stagger Reveal | `translateY(32px)→0` + `opacity 0→1`、各要素80msスタガー | Hero、全セクション見出し |
| 2 | Blur Unveil | `blur(12px)→0` + `scale(0.96)→1` + `opacity 0→1` | Results主要数値 |
| 3 | Slide In Left | `translateX(-60px)→0` + `opacity 0→1` | Features左配置カード |
| 4 | Slide In Right | `translateX(60px)→0` + `opacity 0→1` | Features右配置カード |
| 5 | Scale Up | `scale(0.9)→1` + `opacity 0→1` | Testimonialカード、Reasonsカード |
| 6 | Stroke Draw | SVG `stroke-dashoffset: length → 0` | About Platform Diagram、Flow Timeline |
| 7 | Parallax Drift | スクロール連動の `translateY(±20px)` | 背景SVG装飾（全セクション） |

**prefers-reduced-motion対応:**
`@media (prefers-reduced-motion: reduce)` 検出時、全7パターンを `opacity 0→1`（300ms）のフェードのみに置き換え。Canvasアニメーションは静止画状態に。

## 装飾3層マップ（全セクション × 3層）

> v11の核心。全セクションに最低3層の装飾を配置し、「平坦」から「リッチ」へ転換する。

### Hero

| 層 | 装飾要素 | opacity | 実装 |
|---|---|---|---|
| L1 背景 | グリッドライン（40px間隔） + 右上のラジアルグラデーント（ティールグロー） | 0.04 / 0.15 | CSS `background-image` + `radial-gradient` |
| L2 コンテンツ | バッジのグラデーションボーダー + Connection Canvas（ノード＆ライン） | 0.30 / 1.0 | CSS `border-image` + Canvas API |
| L3 マイクロ | CTAボタンのシマーエフェクト + Canvasノードのマウス追従パルス | 1.0 | CSS `@keyframes shimmer` + JS mousemove |

### Media Trust

| 層 | 装飾要素 | opacity | 実装 |
|---|---|---|---|
| L1 背景 | 薄いティール背景（`#F0F7F8`） | 1.0 | CSS background-color |

## 技術・ライブラリ提案

### 推奨技術（コンセプト実現のための選定）

**1. Vanilla Canvas API（Hero Connection Visualizer + Final CTA Particles）**
「接続」のリアルタイム視覚化。`requestAnimationFrame` + 30fps制限。ノードのグロー描画は `createRadialGradient()` で実装。

**2. IntersectionObserver API（7パターンのスクロールアニメーション）**
`threshold: 0.15` で発火。`data-reveal` 属性のバリエーション（`data-reveal="blur"`, `data-reveal="slide-left"` 等）で7パターンを切り分け。

**3. SVG stroke-dashoffset（About Diagram + Flow Timeline）**
CSS `animation` + IntersectionObserverトリガーで、スクロール進入時に線が描かれる。

**4. CSS Custom Properties + backdrop-filter**
ガラスモーフィズムはHeaderとCTAカードに限定。`backdrop-filter: blur(24px)` + `-webkit-backdrop-filter`。

**5. mousemove イベント（3D Tilt + マグネティックボタン + マウス追従パーティクル）**
Featuresカードの3D Tilt: `perspective(1000px) rotateX(Ydeg) rotateY(Xdeg)`。バネ係数0.08でスムーズに追従。

### 技術的リスク・注意点

| リスク | 影響 | 対策 |
|---|---|---|
| 7パターンアニメーションのパフォーマンス負荷 | モバイルでのFPS低下 | Stagger/Scale/Slideは `will-change: transform, opacity` 設定。同時に表示されるアニメーション数を最大5に制限 |
| Canvas描画のモバイル負荷 | バッテリー消費、発熱 | ノード数をモバイルで8以下に削減。`devicePixelRatio` 上限を2に制限 |
| 3D Tiltのタッチデバイス対応 | タッチでは動かない | タッチデバイスでは3D Tiltを無効化。代わりにタップ時のscale(1.02)パルスを適用 |
| SVG装飾の過剰配置 | DOMノード数増大 | 1セクションあたりのSVG装飾要素は最大3つまで。`aria-hidden="true"` 必須 |
| `prefers-reduced-motion` 対応漏れ | アクセシビリティ違反 | 全CSS animation/transitionに `@media (prefers-reduced-motion: reduce)` を適用。JSアニメーションは `matchMedia` でチェック |

## art-directorへの引き継ぎ

### ビジュアルで実現してほしいこと

1. **「白いのにリッチ」の体感**: 白い背景に3層の装飾（グリッド + SVG幾何学 + マイクロインタラクション）を重ね、「計算された空間」としてのプレミアム感を作る。白=退屈、を完全に覆す

2. **スクロールの旅の多様性**: 各セクションに異なるアニメーションパターンを割り当て、「次はどんな動きが来るんだろう」というワクワク感を作る。7パターンのアニメーション計画を忠実に実装する

3. **Heroの右半分が「接続の実況中継」**: Canvasのノード&ラインは「装飾」ではなく「今この瞬間も動いているUZOUの接続処理」として見えるクオリティ。ノードサイズとグロー強度の増強が必要

4. **ダークセクション（Solution/Final CTA）の奥行き**: `#1F353E` の単色にしない。ノイズテクスチャ + アンビエントグロー + SVG幾何学装飾の3層で立体感を作る

5. **装飾3層マップの完全実装**: 全10セクション × 3層 = 30の装飾要素。各層のopacityを厳密に守り、層間の干渉を避ける

### 絶対に守ってほしいルール

1. **装飾は3層（背景/コンテンツ/マイクロ）に整理し、各層のopacity範囲を守る**。L1: 0.03-0.10 / L2: 0.10-0.40 / L3: 0.40-1.0
2. **`#8BC0CA`（ライトティール）は全体の5%以内**。希少性が輝きを生む
3. **カルーセル/横スクロール禁止**（design-recipe.md準拠）
4. **全アニメーションに `prefers-reduced-motion: reduce` 対応を実装**
5. **WCAG AA コントラスト準拠**。特にダークセクションのテキスト視認性
6. **丸いバブル/ブロブ装飾禁止**。幾何学的・線形の装飾のみ
7. **グラデーション方向は135deg（左上→右下）で統一**
8. **セクションごとに異なるアニメーションパターンを使い分ける**。同じフェードインの繰り返しは不可

### 自由に判断してよい範囲

- 各セクション内のカード具体的レイアウト（ベントグリッドの比率等）
- SVG幾何学装飾の具体的な形状（三角形/六角形/同心円/ドットグリッドの選定）
- グラスモーフィズムの追加適用先（Header/CTAカード以外への展開）
- モバイル時のビジュアル簡略化（Canvas ノード数、SVG装飾の非表示等）
- テクスチャの具体的なパラメータ（ノイズのbaseFrequency、グリッド線のstroke-width等）
- Parallax Driftの移動量（±10px〜±30pxの範囲で判断）
