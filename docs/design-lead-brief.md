**追加問題（AIテンプレ項目外だが致命的）:**

1. **「Geometric Flow」コンセプトの体現度が低い**: ブランドコンセプト「有象無象（形あるものとないものが溶け合う）」がデザインに反映されていない。幾何学パターンはHeroのCanvasパーティクルのみで、以降のセクションでは完全に消える
2. **Heroが弱すぎる**: コンテンツが左半分にのみ集約され、右半分は意図的な空白とされているが、プロフェッショナルなLPとしては「未完成感」に見える。Linear・Stripe・Vercelといった参考サイトは、右半分に強烈なビジュアルがある
3. **ダークセクションの品質不足**: Solution・Flow・Final CTAのダークセクションが `#1F353E` の単色に近く、奥行き・立体感・質感がない。Raycast・Linearが持つ「光と影の演劇的な使い方」が欠けている
4. **数値セクション（Results）がフラット**: 4列の数値が均等に並んでいるだけ。背景装飾・グラデーション境界・視覚的変化がない。数値の「重さ」が感じられない
5. **Testimonialsに人間が存在しない**: 顔写真・ロゴなし。匿名化は理解できるが、「会社名 + 役職 + テキスト」だけでは信頼感が半減する
6. **タイポグラフィの階層が4段階のみ**: display/h1/h2/body/captionの5段階が必要。現状はh1サイズとbodyサイズの差が小さく、視覚的ヒエラルキーが弱い
7. **CTAセクションのコピーが弱い**: 「広告パフォーマンスを次のステージへ。」は凡庸。BtoB SaaS LPとしては予測可能すぎる。記憶に残らない
8. **Heroコピー「メディア広告をもっとスマートに」**: 「もっとスマートに」は2010年代のワードセンス。「AI時代」「次世代」と並ぶ陳腐表現。UZOUならではのコピーではない

## Part 2: 作り直し方針

### 根本問題の特定

v9が「AIっぽく見える」真因は1つ:

> **「Geometric Flow」というコンセプトを宣言したが、実装がコンセプトを体現していない。結果として、よくあるBtoB SaaSテンプレートにパーティクルを乗せたものになっている。**

解決策は「もっと装飾を増やす」ではない。  
**「UZOUというブランドの固有性から生まれるデザイン言語を確立する」**ことが必要。

### デザイン方針: B案「Geometric Flow」をゼロから再構築

B案を捨てる必要はない。ただし再定義が必要。

**v9の問題**: 「Geometric Flow」を「パーティクルのある幾何学的デザイン」と浅く解釈した  
**v10の解釈**: 「有象無象（UZOU）= 形あるものとないものが出会い、溶け合い、循環する」という哲学をデザイン全体で体現する

この解釈の違いが、デザインのあらゆる決定に影響する:

- 背景: 単純なグリッドではなく、「情報のフロー」を感じさせる有機的なライン
- セクション遷移: 白↔ダークの単純切替ではなく、グラデーションが「溶け込む」遷移
- タイポグラフィ: 均一なフォントサイズではなく、「有象無象」らしい対比（極大と極小）
- インタラクション: 「広告とメディアが出会う瞬間」を体験として表現

## Part 3: 品質基準（合格ライン）

### 目標スコア

| 軸 | 目標 | 根拠 |
|---|---|---|
| Design (40) | 32/40以上 | Awwwards Honorable Mention相当 |
| UX/UI (30) | 23/30以上 | CVR最適化 + モバイル完全対応 |
| Creativity (20) | 16/20以上 | BtoB SaaS LPとして「おっ」ポイント3つ以上 |
| Content (10) | 8/10以上 | ブランドコンセプト体現 + コピーの強度 |
| **合計** | **79/100以上** | **Honorable Mention級** |

### 「プロが作った」の証拠として必須の要素

1. **Heroに圧倒的なビジュアルインパクト**: 右半分を「空白」にしない。UZOUのプラットフォームを体験させるインタラクティブな視覚化
2. **各セクションが異なる個性を持つ**: 10セクションが全て異なるレイアウトパターン・背景表現を持つ
3. **ティールの多層的な使い方**: 単色ではなく、深みと光の変化を持つカラーシステム
4. **「有象無象」の哲学が視覚言語になっている**: 特にHero・About・Featuresで明確に体現
5. **モバイルファーストの再設計**: デスクトップの縮小版ではなく、モバイル独自のビジュアル体験

### design-director への指示

**タスク**: 情報設計の再構築と制作進行管理

#### セクション構成の再定義

現行v9の問題: セクション10個が「説明→説明→説明」の線形構造。ユーザーが途中で離脱する

v10で採用すべきストーリーライン（感情の弧を設計する）:

```
[衝撃] Hero
  ↓ 「これは何だ?」と感じさせる
[文脈] Media Trust（マーキー）
  ↓ 「あのメディアも使っているのか」
[共感] Solution（課題）
  ↓ 「そうそう、これで困ってる」
[驚き] Results（数値）
  ↓ 「この規模感は信頼できる」
[理解] About / Platform（UZOUとは）
  ↓ 「なぜこれが実現できるのか」
[納得] Features（強み）
  ↓ 「具体的にこう動くのか」
[確信] Testimonials（声）
  ↓ 「実際に使っている人が言う」
[行動] CTA Strip
  ↓ 「今すぐ問い合わせたい」
[手順] Flow
  ↓ 「導入ハードルは低い」
[疑問解消] FAQ
  ↓ 「もう不安はない」
[決断] Final CTA
```

**変更点**: Solutionをより前面（Hero直下）へ移動。Resultsを感情的な確証として活用

#### 重要なセクション仕様追加

- **Heroビジュアル（必須）**: 右半分にUZOUプラットフォームの「ライブデータ視覚化」風のインタラクティブSVG/Canvasアニメーションを配置。「広告とメディアが出会う瞬間」を抽象的に表現
- **Resultsの演出強化**: 数値4つを単純な4列ではなく、大小の対比をつけた非対称レイアウトに変更。背景にグラデーション境界線を加える
- **CTA Strip廃止・再設計**: 3つのテーブルセルではなく、1つの主CTAを圧倒的に強調するデザインへ

#### 設計書に含めるべき要素

1. セクション構成図（順序・遷移・感情設計）
2. 情報設計（各セクションのH2/本文/CTA構造）
3. コンバージョンファネル設計（どこでダウンロードボタンを表示するか）
4. アセット要件（必要なSVG・CSS・JSパーツ一覧）

### creative-director への指示

**タスク**: 「有象無象」哲学のデザイン言語化

#### コンセプトの再定義

現行ブランドコンセプト（service-profile.mdより）:
- 語源: 「有象無象」= 形あるものとないもの
- Zのロゴに斜め線 → 「嫌われる広告に一石を投じる」意志
- 下部が溶けるデザイン → 「既存概念を溶かしていく」
- 「1」を模した形状 → ナンバーワン

このブランドコンセプトをデザインビジュアルに翻訳せよ:

**「形あるものとないものの接続点」**

- 広告主（有象）とメディア（有象）の間に存在するUZOUは「無象」の存在
- その接続が「流れ」を生む
- 流れが「成果」という形になる

この哲学がHeroのビジュアルからFooterまで一貫して流れることが求められる

#### トーン&マナーの再設定

v9の問題: BtoB SaaS一般のトーン（信頼感・先進性・データ重視）になっており、UZOUの固有性がない

v10で確立すべきトーン:
- **知的だが挑発的**: 「広告は嫌われている。UZOUはそれを変える」という強いスタンス
- **精密だが有機的**: 幾何学的な構造の中に「流れ」がある
- **暗闇の中の光**: ティールの深い暗色をベースに、光のポイントで視線を誘導

#### カラー哲学の確立

ブランドカラー5色をそのまま使うのではなく、「カラーシステム」として再定義せよ:

```
深海（--c-black: #040404, --bg-base: #0a1215）
  ↓ ここから光が生まれる
深淵（--c-dark: #1F353E）
  ↓ UZOUの活動域
流れ（--c-deep: #2B4954）
  ↓ 接続が起きる場所
接触（--c-primary: #34626F）
  ↓ アクション・CTAの色
光（--c-light: #8BC0CA）
  ↓ 結果・成果・成功の色
```

この「深海から光へ」という方向性が、ページを下にスクロールするごとに体験できるようにする

#### ビジュアルモチーフの確立

v10で全セクションに一貫して使うビジュアル言語を定義せよ:

1. **「接続線（Connection Lines）」**: 広告主とメディアをつなぐ線のモチーフ。単純なグリッドではなく、有機的に湾曲した接続線
2. **「ノード（Nodes）」**: 接続点を表す円・菱形の小さな発光点
3. **「フロー（Flow）」**: 左から右へ、または画面外から内側へ流れ込むモーション
4. **「溶解（Dissolve）」**: セクション遷移での「溶け込み」エフェクト

#### コピーの方向性（design-directorと共同で作業）

NG表現（v9で使っている陳腐なワード）:
- 「スマートに」「次世代」「AIが導く」「最適化」「双方を最大化」

GO表現（UZOUらしいワード）:
- 「つなぐ」「流れ」「循環」「接点」「嫌われない広告」「500のメディアが選んだ理由」

Heroコピーの方向性案:
- 案A: 「広告が、メディアを救う。」（逆説的・インパクト重視）
- 案B: 「500のメディアが選んだ、広告の答え。」（社会的証明ファースト）
- 案C: 「有象無象の広告市場に、秩序をもたらす。」（ブランドコンセプトを直接使う）

### art-director への指示

**タスク**: ビジュアルシステムの構築と各セクションの具体化

#### 厳守ルール（絶対に変えるな）

1. Vanilla HTML/CSS/JS のみ（フレームワーク・CDN一切不使用）
2. Google Fonts: Noto Sans JP + Inter のみ
3. ブランドカラー: #8BC0CA / #34626F / #2B4954 / #1F353E / #040404
4. ファイル構成: index.html / style.css / script.js の3ファイルのみ
5. prefers-reduced-motion 対応（全アニメーション）
6. WCAG AA コントラスト準拠

#### セクション別ビジュアル仕様

**01. Header**
- v9と同様のガラスモーフィズムナビ（品質十分）
- 変更点: スクロール時の背景をより透明感のある `rgba(10,18,21,0.85)` + `backdrop-filter: blur(24px)` に調整

**02. Hero（最重要・最も時間をかけること）**

問題: v9はコンテンツが左半分のみ。右半分が空虚
解決: 右半分に「UZOU Connection Visualizer」を配置

具体的なビジュアル:
- Canvas上に複数の「ノード」（広告主・メディア）が浮遊
- ノード間をベジェ曲線の「接続線」が走る
- アクティブになった接続点が光る（ティールのグロー）
- マウスホバーで最寄りのノードが引き寄せられる（マグネティック効果）
- これが「広告とメディアが出会う瞬間」の視覚化

コピー: creative-directorが確定したものを採用。長さはH1で7語以内厳守
Hero背景: 現行のグリッドラインを維持しつつ、右上と左下に「アンビエントグロー」を強化（現行の2倍の存在感）

**03. Media Trust（マーキー）**
- v9のマーキーは品質十分
- 変更点: 2行マーキーを維持しつつ、セクション背景に微細なグラデーション境界を追加（Hero → About への「溶解」遷移）

**04. Solution（課題）**
- v9問題: Solutionが中間にあり、ユーザーが課題に共感する前にAboutを読まされる
- v10: MediaTrust直後に配置
- レイアウト変更: 2列均等ではなく、「タブ切替」または「トグル」で Advertisers / Publishers を切り替える形式
- なぜ: 2列均等は情報量が多すぎてスキャンしにくい
- ビジュアル: 各課題項目に、解決を暗示する小さなSVGアイコンを追加

**05. Results（実績）**
- v9問題: 4列均等で単調
- v10変更: 非対称レイアウト
  - 主要KPI（月間広告リクエスト50億+）を画面左に大きく
  - 残り3つのKPIを右側に縦積み
  - 背景: 深いダークティールに「データの流れ」を表すSVGラインアニメーション
  - 数値フォントサイズ: 主KPIは 80-100px、サブKPIは 56-64px

**06. About（UZOUとは）**
- v9問題: 左の縦書き「UZOU」デコテキストが陳腐
- v10変更: 縦書き装飾を廃止
- 新レイアウト: フルワイドの「Platform Diagram」
  - 画面中央にUZOUプラットフォームを大きく
  - 左右から広告主・メディアが「流れ込む」インタラクティブSVGダイアグラム
  - スクロール連動でラインが描画される

**07. Features（強み）**
- v9のベントグリッドは方向性正しい
- 変更: カード内のビジュアルをより強化
  - Feature-01（AI最適化）: リアルタイムで棒グラフが変動するアニメーション（setIntervalで値が変わる）
  - Feature-02（メディアネットワーク）: ノードが増殖するSVGアニメーション
  - Feature-03（ワンストップ）: フロー図の各ステップが順次ハイライトされるアニメーション

**08. Testimonials（声）**
- v9問題: 顔写真なし、3カード均等、信頼感が弱い
- v10変更: 
  - 均等3カードをやめ、中央1枚を大きくしたフィーチャー形式
  - カードに会社ロゴ（テキストロゴ）を追加
  - 数値（「CPA 30%改善」）をカード内に大きく表示
  - 背景: 薄いティールグラデーション（#F0F7F8 → #E8F2F4）

**09. Flow（導入の流れ）**
- v9の縦タイムラインは品質良好
- 変更: STEPドットを小→大のサイズ変化で「現在のステップ」感を演出
- ホバーで各STEPの詳細が展開するインタラクション追加

**10. Reasons（選ばれる理由）**
- v9の非対称ベントグリッドは方向性正しい
- 変更: 大カードのSVG図解をよりリッチに（現行は線とテキストのみ → ノードとフロー線のビジュアル）

**11. FAQ**
- v9は品質良好
- 変更: 左固定見出し部分に「UZOU」のブランドカラーグラデーションを追加

**12. Final CTA**
- v9問題: コピーが弱い、グラデーション背景が単調
- v10変更: 
  - 背景: `#34626F` → `#2B4954` → `#1F353E` の3段グラデーション + ノイズテクスチャ
  - インタラクティブパーティクル: マウスに追従する光の粒子
  - コピー: creative-directorが確定したものを採用

#### タイポグラフィシステム（5段階）

```
Display: Inter 900 / 72-96px / letter-spacing: -0.05em
H1:      Inter 800 / 52-64px / letter-spacing: -0.04em
H2:      Inter 700 / 36-44px / letter-spacing: -0.03em
Body:    Noto Sans JP 400 / 16px / line-height: 1.9
Caption: Inter 500 / 12-13px / letter-spacing: 0.12em / uppercase
```

#### アニメーションシステム

**スクロールアニメーション（全セクション必須）:**
- フェードアップ: `translateY(32px) → 0 + opacity 0→1`
- フェードインレフト: `translateX(-48px) → 0 + opacity`
- スケールイン: `scale(0.95) → 1 + opacity`
- ブラーリビール: `blur(12px) → 0 + opacity`（Resultsの数値に使用）
- スタッガー: 同一セクション内で 80ms の時間差

**マイクロインタラクション:**
- ボタンホバー: `translateY(-3px) + box-shadow拡大 + shimmer`（現行維持・強化）
- カードホバー: `translateY(-6px) + border-color変化 + shadow拡大`
- ナビリンク: 下線アニメーション（現行維持）
- CTAパルス: 3秒周期のグロー（現行維持・強化）

**Canvasパーティクル（Hero右半分）:**
- ノード数: 12-18個（広告主側 6-8、メディア側 6-8、UZOU中心 1）
- ノード間接続: 距離200px以内でベジェ曲線を描画
- アクティブアニメーション: ランダムに接続が光る（3-5秒周期）
- マウスインタラクション: ホバーで最寄りノードが引き寄せられる

## Part 6: 禁止事項（絶対に使わないもの）

| カテゴリ | 禁止内容 | 理由 |
|---|---|---|
| コピー | 「スマートに」「次世代」「AIが導く」「最適化」「双方を最大化」 | 陳腐・AI感・競合と区別がつかない |
| デザイン | Bootstrap/MUIのデフォルトコンポーネントのまま使用 | テンプレ感 |
| デザイン | 3色以下のカラー | AIテンプレ |
| デザイン | 全セクション同じpadding | リズムの欠如 |
| デザイン | ホバーで色変化のみ | インタラクション品質不足 |
| デザイン | 全カード均等サイズ | 視覚的メリハリの欠如 |
| 技術 | CDN経由のライブラリ | 技術制約違反 |
| 技術 | 実際の管理画面スクリーンショット | 会議決定事項 |
| 技術 | 丸いバブル/ブロブ装飾 | ブランド方針（幾何学・線形指定） |
| 技術 | カルーセル/横スクロール | モバイルUX悪化 |
